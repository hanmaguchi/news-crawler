# 사용 가이드

## 앱 실행

```bash
cd news-crawler
uv run streamlit run app.py
```

브라우저에서 `http://localhost:8510` 접속.

## 검색어 입력 방식

### 기본 — 단일 키워드
```
인공지능
```

### AND 검색 — 두 단어가 모두 포함된 기사
띄어쓰기로 구분하면 두 단어가 **모두** 포함된 기사만 수집한다.
```
비트코인 ETF
```
→ "비트코인"과 "ETF"가 같은 기사 어딘가에 둘 다 있으면 포함.
단어 순서나 위치는 상관없다 (같은 문장일 필요도 없음).

### 구문 검색 — 정확히 이 문구가 붙어있는 기사
큰따옴표로 감싸면 해당 문구가 **그대로 붙어있는** 기사만 수집한다.
```
"비트코인 ETF"
```
→ "비트코인 ETF"라는 정확한 구문이 포함된 기사만.

### OR 수집 — 쉼표로 여러 키워드 각각 검색 후 합치기
쉼표로 구분하면 키워드별로 **별개의 검색**을 실행한 뒤 결과를 합치고 중복을 제거한다.
```
삼성전자, 이재용, 갤럭시
```
→ 각 키워드에 해당하는 기사를 모두 수집. 하나의 기사에 여러 키워드가 포함되면 1건으로 합산.

## 기본 사용법

### 1단계 — 검색어 입력
위의 검색어 입력 방식 참고.

### 2단계 — 최근 검색어 활용 (히스토리)

검색을 진행할 때마다 검색어가 자동으로 기록된다(최대 10개).

- **최근 검색어** 드롭다운에서 이전 검색어를 선택하면 입력란에 자동으로 채워진다
- 같은 키워드를 반복 모니터링할 때 편리하다
- 히스토리는 브라우저 세션 동안 유지된다

### 3단계 — 기간 설정

**시작일**과 **종료일**을 날짜 선택기로 지정한다.

- 기본값: 오늘 기준 최근 7일

> **참고 (네이버)**: 네이버 API는 기간 범위 파라미터를 공식 지원하지 않는다.
> 최신순으로 받아오면서 기간을 필터링하므로 최대 ~1,000건까지만 수집된다.

### 4단계 — 출처 선택

- **네이버**: `.env`에 API 키가 설정된 경우에만 활성화됨
- **구글**: 키 없이 항상 사용 가능
- **커스텀 RSS**: 하단 섹션에서 RSS URL 추가

### 5단계 — 검색 실행

**🔍 검색** 버튼 클릭. 키워드·소스별 수집 진행 상태가 표시된다.

### 6단계 — 결과 확인

#### 📈 트렌드 차트
날짜별 기사 수를 막대 차트로 보여준다. 이슈가 언제 가장 많이 보도됐는지 한눈에 확인 가능.

#### 감성 요약
수집된 기사 제목을 **snunlp/KR-FinBert-SC** 딥러닝 모델로 분류해 긍정 / 부정 / 중립 건수를 표시한다.
한국 금융·경제 뉴스에 특화된 모델로, 키워드 방식보다 맥락 이해 정확도가 높다.

> 최초 실행 시 모델(~440MB)을 다운로드한다. 이후 실행은 캐시에서 즉시 로드.

#### 언론사 필터
수집 후 특정 매체만 추려볼 수 있다. 모든 내보내기(엑셀·CSV·JSON)에 필터가 적용된다.

#### 언론사 블랙리스트
검색 전 **🚫 언론사 블랙리스트** 섹션에서 제외할 매체를 등록하면 결과에서 자동 제외된다.
세션 동안 유지되며, 해제 버튼으로 언제든 복구 가능하다.

#### 결과 테이블

웹 화면과 엑셀 파일 모두 동일한 열 구성을 사용한다.

| 열 | 설명 |
|----|------|
| # | 순번 |
| 배포일자 | 기사 발행 일시 |
| 언론사 | 발행 매체명 |
| 제목 | 기사 제목 (검색어 부분 **노란 하이라이트**) |
| 링크 | **기사보기** 클릭 시 원본 이동 |
| 감성 | 🟢 긍정 / 🔴 부정 / ⚪ 중립 |
| 출처 | 수집 엔진 (예: `네이버`, `구글`, `네이버, 구글`) |

### 7단계 — 내보내기

파일명 형식: `news_검색어_시작일_종료일.xlsx / .csv / .json`

| 버튼 | 포맷 | 용도 |
|------|------|------|
| 📥 엑셀 다운로드 | `.xlsx` | 색상·하이퍼링크 포함, 오피스 작업용 |
| 📄 CSV 다운로드 | `.csv` | 데이터 분석 도구 (파이썬·R·스프레드시트) |
| 📋 JSON 다운로드 | `.json` | 개발·API 연동용 |

> CSV는 UTF-8 BOM으로 저장되어 한글 깨짐 없이 Excel에서 바로 열린다.

#### 엑셀 파일 구성

- 컬럼: **순번 / 배포일자 / 언론사 / 제목 / 링크 / 감성 / 출처**
- 감성 셀: 긍정(연초록) / 부정(연빨강) 배경색
- 링크 셀: `기사보기`만 표시, 마우스 올리면 실제 URL 툴팁
- 배포일시: `yyyy-mm-dd hh:mm` 형식
- 언론사 필터 적용 시 필터된 결과만 저장

## 커스텀 RSS 소스 추가

1. **➕ 커스텀 RSS 소스 추가** 섹션 펼치기
2. 소스 이름 + RSS URL 입력 후 **추가**
3. 이후 검색 시 해당 피드도 함께 수집됨

> 세션 중에만 유지됨. 앱 재시작 시 초기화.

### 주요 RSS 피드 예시

| 매체 | RSS URL |
|------|---------|
| 연합뉴스 | `https://www.yonhapnews.co.kr/RSS/headline.xml` |
| 한겨레 | `https://www.hani.co.kr/rss/` |
| 경향신문 | `https://www.khan.co.kr/rss/rssdata/kh_news.xml` |
| 한국경제 | `https://www.hankyung.com/feed/all-news` |
| ZDNet Korea | `https://zdnet.co.kr/rss.asp` |

## 자주 묻는 질문

**Q. 비트코인과 ETF가 같은 기사에 있는 걸 찾으려면?**
A. `비트코인 ETF`(띄어쓰기)로 입력하면 된다. 두 단어가 반드시 붙어있을 필요는 없고, 같은 기사 어딘가에 둘 다 있으면 포함된다. 정확히 붙은 구문을 찾으려면 `"비트코인 ETF"`처럼 큰따옴표로 감싼다.

**Q. 쉼표 구분과 띄어쓰기 구분의 차이는?**
A. 띄어쓰기(`비트코인 ETF`)는 두 단어가 모두 있는 기사를 수집하는 AND 검색이다. 쉼표(`비트코인, ETF`)는 각각을 별개로 검색한 뒤 합치는 OR 수집이다.

**Q. 감성 분류가 부정확해 보여요.**
A. 키워드 사전 방식이라 맥락을 파악하지 못한다. "주가 하락 우려 해소" 같은 제목은 부정으로 잘못 분류될 수 있다. 참고용으로만 활용하길 권장한다.

**Q. 네이버 체크박스가 비활성화되어 있어요.**
A. `.env`에 `NAVER_CLIENT_ID` / `NAVER_CLIENT_SECRET`이 없는 경우다. [설치법.md](./설치법.md) 참고.

**Q. 구글 링크를 클릭하면 구글 뉴스 페이지로 가요.**
A. 정상 동작이다. 구글 News RSS는 리다이렉트 URL을 제공하며, 클릭하면 실제 언론사 페이지로 이동한다.

**Q. 히스토리가 사라졌어요.**
A. 세션 중에만 유지된다. 앱 재시작 또는 새로고침 시 초기화된다.

**Q. 수집 건수가 적게 나와요.**
A. 네이버 최대 ~1,000건, 구글 RSS는 최신 기사 위주다. 기간을 줄이거나 키워드를 구체화하면 더 정확한 결과를 얻을 수 있다.
