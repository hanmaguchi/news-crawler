# 사용 가이드

## 앱 실행

```bash
cd news-crawler
uv run streamlit run app.py
```

브라우저에서 `http://localhost:8510` 접속.

## 기본 사용법

### 1단계 — 검색어 입력

**검색어** 입력란에 원하는 단어나 구문을 입력한다.

- 단일 키워드: `삼성전자`
- 구문 검색: `인공지능 규제`
- 복합 키워드: `전기차 배터리`

### 2단계 — 기간 설정

**시작일**과 **종료일**을 날짜 선택기로 지정한다.

- 기본값: 오늘 기준 최근 7일
- 원하는 날짜를 클릭해 변경한다

> **참고 (네이버)**: 네이버 API는 기간 범위 파라미터를 공식 지원하지 않는다.
> 최신순으로 기사를 받아오면서 지정 기간 범위에 해당하는 기사만 필터링하므로,
> 기간이 길거나 검색량이 많은 키워드는 최대 1,000건까지만 수집된다.

### 3단계 — 출처 선택

- **네이버** 체크박스: `.env`에 API 키가 설정된 경우에만 활성화됨
- **구글** 체크박스: 키 없이 항상 사용 가능

둘 다 체크하면 두 출처를 동시에 수집한 뒤 중복을 자동으로 제거한다.

### 4단계 — 검색 실행

**🔍 검색** 버튼을 클릭한다. 소스별로 수집 진행 상태가 표시된다.

### 5단계 — 결과 확인

수집 완료 후 결과 테이블이 표시된다:

| 열 | 설명 |
|----|------|
| 제목 | 기사 제목 |
| 언론사 | 발행 매체명 |
| 배포일자 | 기사 발행 일시 |
| 출처 | 어느 엔진에서 수집했는지 (예: `네이버`, `구글`, `네이버, 구글`) |
| 링크 | **기사보기** 클릭 시 원본 기사로 이동 |

- 결과는 **최신순**으로 정렬된다
- 두 출처에서 동일 기사가 잡히면 **1건으로 합쳐지고** 출처에 둘 다 표기된다 (예: `네이버, 구글`)

### 6단계 — 엑셀 다운로드

**📥 엑셀 다운로드** 버튼을 클릭하면 `.xlsx` 파일이 저장된다.

파일명 형식: `news_검색어_시작일_종료일.xlsx`
(예: `news_삼성전자_2025-06-12_2025-06-19.xlsx`)

#### 엑셀 파일 구성

- 컬럼: **제목 / 언론사 / 배포일자 / 출처 / 링크**
- 1행 고정 (스크롤해도 헤더가 보임)
- **링크 셀**: 셀에는 `기사보기`만 표시, 마우스를 올리면 실제 URL이 툴팁으로 표시됨
- 배포일자 형식: `yyyy-mm-dd hh:mm`

## 커스텀 RSS 소스 추가

네이버·구글 외에 다른 출처를 추가하고 싶을 때 사용한다.

1. **➕ 커스텀 RSS 소스 추가** 섹션을 클릭해 펼친다
2. **소스 이름** 입력 (예: `연합뉴스`)
3. **RSS 피드 URL** 입력 (예: `https://www.yonhapnews.co.kr/RSS/headline.xml`)
4. **추가** 버튼 클릭 → 출처 목록에 등록됨
5. 이후 **🔍 검색** 시 해당 피드도 함께 수집됨
6. 등록된 소스 옆 **삭제** 버튼으로 제거 가능

> 커스텀 RSS는 세션 중에만 유지된다. 앱을 재시작하면 초기화된다.

### 주요 RSS 피드 예시

| 매체 | RSS URL |
|------|---------|
| 연합뉴스 | `https://www.yonhapnews.co.kr/RSS/headline.xml` |
| 한겨레 | `https://www.hani.co.kr/rss/` |
| 경향신문 | `https://www.khan.co.kr/rss/rssdata/kh_news.xml` |
| 한국경제 | `https://www.hankyung.com/feed/all-news` |
| ZDNet Korea | `https://zdnet.co.kr/rss.asp` |

## 자주 묻는 질문

**Q. 네이버 체크박스가 비활성화되어 있어요.**
A. `.env` 파일에 `NAVER_CLIENT_ID`와 `NAVER_CLIENT_SECRET`이 설정되지 않은 경우다.
[설치법.md](./설치법.md)의 **네이버 API 키 발급** 항목을 참고한다.

**Q. 구글 링크를 클릭하면 구글 뉴스 페이지로 가요.**
A. 구글 News RSS는 `news.google.com` 리다이렉트 URL을 제공한다. 클릭하면 실제
언론사 기사 페이지로 자동 이동하므로 정상 동작이다.

**Q. 중복이 완전히 제거되지 않는 것 같아요.**
A. 제목이 출처마다 약간 다르게 표기되는 경우(예: 띄어쓰기 차이, 기호 차이)
동일 기사로 인식되지 않을 수 있다. 정규화 알고리즘의 한계다.

**Q. 수집 건수가 적게 나와요.**
A. 네이버는 최대 ~1,000건 제한이 있고, 구글 RSS는 최신 기사 위주로 제공한다.
기간을 줄이거나 키워드를 구체화하면 원하는 기사를 더 정확하게 수집할 수 있다.
