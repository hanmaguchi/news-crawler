# 뉴스기사 크롤러 기획서

## 1. 배경 및 목적

미디어 모니터링, 경쟁사 동향 파악, 리서치 업무 등에서 특정 키워드의 뉴스를
수집하는 작업은 반복적으로 발생한다. 기존에는 포털 사이트에서 수동으로 검색하고
결과를 복사·붙여넣기 하거나, 유료 미디어 모니터링 서비스에 의존했다.

이 도구는 **무료·공개 API / RSS를 활용해 복수의 출처에서 뉴스를 자동 수집하고
엑셀로 정리**함으로써 반복 수작업을 제거한다.

---

## 2. 핵심 요구사항

| # | 요구사항 | 구현 방식 |
|---|----------|-----------|
| 1 | 검색어 입력 | Streamlit 텍스트 입력칸 |
| 2 | 기간(시작·종료) 설정 | 날짜 선택 위젯 (기본: 최근 7일) |
| 3 | 네이버·구글 기본 출처 | 네이버 검색 API + 구글 News RSS |
| 4 | 추가 출처 확장 | 커스텀 RSS URL 등록 UI |
| 5 | 제목 / 언론사 / 배포일자 / 링크 표시 | 결과 테이블 + 엑셀 출력 |
| 6 | 링크: 셀에는 미노출, 호버 시 URL 표시 | 엑셀 하이퍼링크 + 툴팁 |
| 7 | 엑셀 다운로드 | `.xlsx` 버튼 다운로드 |
| 8 | 중복 기사 제거 | 정규화 제목 기반 dedup + 출처 병합 |

## 3. 수집 출처 설계

### 3-1. 네이버 검색 API
- 엔드포인트: `https://openapi.naver.com/v1/search/news.json`
- 인증: Client ID / Client Secret (무료, 하루 25,000건)
- 특징: 날짜 범위 파라미터 없음 → `sort=date`로 최신순 페이지네이션 후 기간 필터링
- 언론사명: API 미제공 → 원본 URL 도메인으로 추정 (도메인-매체명 매핑 테이블 + 폴백)
- 최대 수집: ~1,000건/검색

### 3-2. 구글 News RSS
- 엔드포인트: `https://news.google.com/rss/search?q=…&hl=ko&gl=KR&ceid=KR:ko`
- 인증: 불필요 (무료 공개)
- 기간 필터: 쿼리에 `after:YYYY-MM-DD before:YYYY-MM-DD` 연산자 사용
- 링크: `news.google.com` 리다이렉트 URL (클릭 시 정상 이동)
- 제목: 구글이 붙이는 "– 매체명" 접미사를 자동 제거

### 3-3. 커스텀 RSS
- 임의의 RSS 피드 URL을 이름과 함께 UI에서 등록
- 키워드·날짜는 피드 수신 후 클라이언트에서 필터링

## 4. 중복 제거 전략

네이버와 구글이 같은 기사를 서로 다른 URL(리다이렉트 포함)로 반환하기 때문에
URL 단순 비교만으로는 중복 탐지가 불충분하다.

**2단계 dedup:**
1. **정규화 제목 키**: 제목을 소문자로 변환 후 공백·구두점·특수문자 제거 → 동일하면 같은 기사
2. **URL 정확 일치**: 위를 통과한 항목 중 URL이 동일한 것 제거

충돌 시 먼저 수집된 소스 우선 유지, `source` 열에 중복 출처를 병합해 표기
(예: "네이버, 구글").

## 5. 아키텍처

```
app.py (Streamlit UI)
    │
    ├── crawler/sources/
    │   ├── base.py       NewsSource ABC  ← 새 엔진 추가 지점
    │   ├── naver.py      네이버 API
    │   ├── google.py     구글 RSS
    │   └── rss.py        범용 RSS
    │
    ├── crawler/dedup.py  중복 제거
    ├── crawler/export.py 엑셀 출력 (openpyxl)
    └── crawler/models.py Article 데이터클래스
```

**확장 방법**: `NewsSource`를 구현하고 `app.py` 소스 목록에 추가하면 새 검색엔진을
붙일 수 있다. RSS 기반 추가는 UI에서 코드 수정 없이 가능하다.

## 7. 제약사항 및 한계

| 항목 | 내용 |
|------|------|
| 네이버 기간 필터 | API 미지원 → 최신순 스캔 후 클라이언트 필터 (최대 ~1,000건) |
| 네이버 언론사명 | 도메인 추정, 미등록 도메인은 호스트명 그대로 표시 |
| 구글 링크 | news.google.com 리다이렉트 URL (최종 언론사 URL 자동 해석 미포함) |
| 구글 HTML 스크래핑 | 미구현 (ToS 위반·차단 위험) |
| 기사 본문 | 메타데이터만 수집, 전문 크롤링 미포함 |

## 8. 향후 개선 방향 (범위 밖)

- 구글 RSS 리다이렉트 URL → 최종 언론사 URL 자동 해석
- 기사 본문 요약 (LLM 연동)
- 키워드 알림 (이메일 / Slack)
- 데이터베이스 저장 및 히스토리 조회
